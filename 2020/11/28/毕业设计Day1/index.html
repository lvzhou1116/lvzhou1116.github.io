Day1 调研薪资管理系统 + Java Web学习

1 将要使用到的Java Web技术

技术框架：SSM（SpringMVC+Spring+Mybatis）+Mysql

前端：Bootstrap+Jquery+Ajax+Jsp+HTML

2 常见薪资管理系统的特征

2.1 使用者

  • 员工：具有自己的用户名和密码，可以查询到自己的工资单
  • 主管部门：具有自己的用户名和密码，对员工信息进行增删改查和报表的汇总、打印、导入、导出等操作
  • 系统管理员：具有唯一的用户名和密码，可对数据进行备份等操作

2.2 功能

  • 员工信息管理：实现对员工基本信息的管理操作，包括员工基本数据的增删改查功能
  • 工资管理：实现对基础数据的设定（如工龄工资、岗位工资、绩效的计算规则、所得税率等），工资的统计、发放，工资条的打印等
  • 工资查询：根据条件查询员工的历史工资记录（如按月查询、按姓名查询、按工号查询、按部门查询等）
  • 汇总打印：提供报表的打印输出功能，可以报表的形式打印员工信息、工资发放报表、工资历史报表等
  • 系统维护：进行数据的备份和恢复，并实现数据的导入和导出
  • 管理员设置：实现创建或删除系统用户、更改当前系统用户的密码等功能

2.3 大致用例

员工：登录、工资查询、退出登录

主管部门：登录、员工信息管理、工资的基本设定、汇总打印、退出登录

系统管理员：登录、系统用户管理、系统维护、退出登录

3 数据表的大致设计

具体数据表的设计需要经过需求分析后，画E-R图来设计，这里只做出粗略介绍。

员工基本信息表（包括ID、工号、密码、姓名、部门等个人基础信息）

工资表（包括ID、工号、姓名、基础工资、绩效奖金、发放时间等信息）

系统用户信息表（包括USR_ID、用户编号、用户类型、用户密码等信息）

员工银行账户表（包括ID、工号、姓名、银行卡号等信息）

……

4 具体实现相关事宜

以往做Web应用项目，用原生的JSP+Servlet+JDBC即可，但用这种方法做出来的项目，其拓展性和可维护性不足，所以自己在掌握原生知识的基础上还得花时间深入学习SSM框架，以做出更好的项目。此外，需要花比较长一段时间进行需求分析，自己还要好好回顾数据库设计的知识，以设计出条理清晰、查询高效的数据表。

5 今天的自学内容

  • JSP作用域
  • EL表达式
  • JSTL
  • 过滤器Filter
