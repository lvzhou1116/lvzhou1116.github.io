3 项目的构建

核心思想：按**业务**进行代码的编写。

3.1 登录业务

首先编写好login的前端页面，这里就不再展示前端代码了。

前端页面如下：

登录者有两类，读者和管理员，所以在数据库中建立两张表，用于存放读者和管理员的信息（如id，账号、密码、卡号、联系方式等）。

使用MySQL建立library数据库，在library数据库中添加两张表，reader表和bookadmin表。

reader表:

id username password name tel cardid gender

bookadmin表:

id username password

现在开始编写Servlet，用来处理前端页面传来的数据。

在src文件夹下新建com.lz.controller包，在该包中新建LoginServlet，该类继承HttpServlet，重写其doPost请求。

因为controller会将请求转发给service，所以需要新建com.lz.service包。在该包中新建接口LoginService（其命名与业务相关），该接口的登录方法返回值需要是数据库中查出的Reader/bookadmin信息，所以我们需要创建com.lz.entity包，在该包中创建Reader实体类，其私有成员变量与数据表的字段一一对应。

现在我们回头看看com.lz.service包中的LoginService接口，其登录方法login的返回值类型现在可以确定为Reader类，参数为用户名和密码。

在com.lz.service中新建impl包，该包存放接口的实现类，新建LoginServiceImpl类，实现LoginService接口，在LoginServiceImpl类中实现LoginService接口的login方法。

接下来创建com.lz.repository包，在包中创建ReaderRepository接口（其命名与数据表名称相关），在该接口中创建login方法，返回值为Reader，参数为用户名和密码。

在com.lz.repository中新建impl包，该包存放接口的实现类，新建ReaderRepositoryImpl类，实现ReaderRepository接口，在ReaderRepositoryImpl类中实现ReaderRepository接口的login方法。

这样，MVC的整体流程思路就走通了，浏览器(V)——》controller(C)——》service(M)——》repository(M)——》数据库，以实体类Reader(M)作为数据传输的对象。此时，项目结构如下：

思路接着往回走，来到最上层的controller，在LoginServlet类的doPost方法中编写代码获取reader对象，并加以判断，若为null，则登陆失败。

在当前情况下，已经可以验证读者的登录，下面开始验证管理员的登录。

首先，第一个想法是在LoginServlet的doPost方法中加一个switch语句用于区分reader和admin，但业务的处理往往是由service完成，所以我们在LoginService接口的login方法中加一个String type参数，然后接着在LoginServiceImpl类的login方法中添加String type参数。

接着，在LoginServiceImpl类的login方法中添加switch语句，如果type为“admin”，就要返回一个admin对象，但实体类中没有Admin，所以我们要先去entity包中编写Admin类（私有成员与数据表的字段一一对应）。

Admin类编写完后，回到LoginServiceImpl类的login方法，发现login方法的返回值是Reader，无论如何都是返回不了Admin的，所以将login方法的返回值改成Object（多态）。

由于admin信息存在bookadmin表里，并不是之前的reader表，根据repository与表一一对应的原则，我们需要在Repository包中创建一个AdminRepository接口，并写出它的实现类AdminRepositoryImpl（与之前类似）,AdminRepositoryImpl类的login方法与ReaderRepositoryImpl类非常相似，改一下sql语句和一些细节即可。

最后修改LoginServiceImpl类中的login方法，修改LoginService接口，修改LoginServlet类的doPost方法即可。
